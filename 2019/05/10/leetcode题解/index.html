leetcode题解

Posted on 2019-05-10 | Edited on 2019-05-16 | In Leetcode

为什么要做题呢？其实做题只是为了让你保持一种编程的状态，保持一种每天解决问题的习 惯，因为编程其实就是解决实际问题的过程啦。每天一点点，其实不会花费太多的时间啊，以下我将分类记下一些leetcode的题解。因为有很多这样的博客了，而且也分类写的很好了，我想做的只是记录一下我的历程啦，在其中也参照了很多他人的博客，分类是参考某博客的分类。

数据结构与算法

LinkedList

常用的解题方法

  • 遍历链表
  • 利用递归的思想，如反转链表
  • 双指针：快慢指针

Remove Duplicates from Sorted List

思路：遍历链表，在遍历时判断当前结点的值与后面结点的值是否相等，时间复杂度O(n)。

Remove Duplicates from Sorted List II

思路：采用双指针，一个指针指向需要保留的结点，一个指针对有重复值的结点进行查找。

Linked List Cycle

思路：采用快慢指针的思想。如果存在环，快指针终归会追上慢指针

string

常用的解题方法

  • 用hash数组进行解决
  • 双指针

Implement strStr()

思路：使用暴力破解：主串需要进行回溯，或者使用KMP算法：主串不需要进行回溯。

Valid Anagram

思路：创建一个hash数组进行标记。

Valid Palindrome

思路：采用双指针的思想，在这当中，使用了java提供的一些函数。

stack

需要掌握的思想

  • 模拟进出栈
  • 用栈解决问题的技巧

Min Stack

思路：用两个栈，一个栈为最小值栈，一个栈为真实数据栈。

Array

常用的解题方法

  • 遍历数组，标记变量
  • 双指针

Remove Duplicates from Sorted Array

思路：使用双指针，一个指针为遍历指针；另一个指针指向结果数组的每个位置，判断该位置要填充的元素。

Remove Duplicates from Sorted Array II

思路：同上题的思路相同，只是多一个变量count对元素的数量进行统计。

Remove Element

思路：双指针，在遍历时用另一个指针确定结果数组的每一个位置的最终值

Majority Element

思路：设置变量majority存储可能是众数的元素，设置一个计数器count对元素的出现次数进行计数，在遍历数组的过程操作count变量进行一些操作。

Search a 2D Matrix II

Search a 2D Matrix
思路：根据题目的条件可知数组是有规律的，按照规律以某个元素所在行和列进行搜索，而减少对整个数组进行搜索。每个元素的行和列构成递增序列。

Move Zeroes

思路：先将前面非零数字进行填充，将数组之后空的位置填充上零。
