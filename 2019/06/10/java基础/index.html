Posted on 2019-06-10 | Edited on 2019-06-29 | In JAVA

经济基础决定上层建筑，打好基础，我们才能走得更远。以下是JAVA基础知识的总结，从概念的层面进行了讲述。我们为什么常常会觉得难呢？有很多时候是因为我们不解其意，一些专业的术语加深了我们理解的难度。理解了，才能记忆得更多。有一种方法可以检测我们是否正真的理解，那就是向别人讲述，这也就是费曼学习法。还有你应该经常巩固，去复习，找到适合自己的学习方法，制定合理的目标。

异常分类及处理

  • 概念
    • 某个方法不能正常执行，则可以通过另一种路径退出。
    • 特点：阻碍接下来代码的执行。
    • API异常类的设计。
  • 异常的分类
    • Error
    • Exception(分为RuntimeException(运行时异常)和CheckedException(编译时会进行检查))
  • 异常的处理方式
    • throw（函数内部、抛出异常对象）、throws（可抛出多个异常）：功能不同（抛出具体的异常和声明式异常（可能会发生异常））消极处理，抛出由调用者惊醒处理。
    • 系统自动抛出异常
    • try…catch…finally..捕获异常并进行处理,finally语句无论如何都会执行，一般用于释放资源，如在JDBC当中用于最后的连接资源的释放。

注解

  • 概念
    • 可以把注解理解为代码里的特殊标记，这些标记可以在编译，类加载，运行时被读取，并执行相应的处理。Annotation是接口，程序可通过Annotation对象获取到注解当中的元数据信息。
    • 元程序：描述程序的程序，元数据：关于数据的数据。（描述数据的数据）
  • 元注解（4种 用于自定义注解）
    • 修饰的对象范围（@Target） 类型成员（方法、构造函数、成员变量） 类
    • 定义被保留的时间长短（@Retention） （source\class\runtime）
    • 文档化（@Documented）
    • @Inherited（用该元注解修饰的注解被用于一个class,则注解也被用于该class子类）
  • java提供的标准的注解
    • @Override @Deprecated
  • 注解处理器
    • 作用：读取注解。
    • 步骤：1.自定义注解 2.使用注解 3.通过注解处理器获取信息（通过反射获取要处理的注解、处理注解）

反射

  • 概念
    • 动态语言：在运行时可以动态地改变其结构。
    • 反射的含义：在运行时可以获取到类和对象的所有属性、方法、构造函数。
  • 应用场景
    • 编译是没办法获取到对象的真实类型（比如向上转型），只有在运行时才能得到对象的具体类型，
    • 运行时动态加载外部的类，这时需要用到反射的机制，获取类的属性和方法。
  • 反射API
    • 4个类（Class、Field、Method、Constructor）
  • 使用
    • 获取Class对象，本质（编译后的字节码）（获取方法：通过对象（getClass()方法）,类（class属性），Class类的静态方法forName(“全类名”)）
    • 获取属性、方法、构造函数（getDeclaredMethods\Field\Contructor）
    • 创建对象（Class对象和Constructor对象的newInstance()方法）
